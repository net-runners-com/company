# 土屋建築株式会社 ホームページ制作 要件定義書

**作成日**: 2026-04-01  
**作成者**: タクト（リサーチ部門）  
**プロジェクト予算**: 200,000円（税別）

---

## 1. プロジェクト概要

土屋建築株式会社のコーポレートサイトを新規制作する。  
中小規模の建築会社として、**地域密着・信頼感・施工実績の訴求**を軸に、問い合わせ獲得を主目的とするサイトを構築する。

### 想定ターゲット
- 住宅の新築・リフォームを検討している個人（30〜60代）
- 法人の建築工事を発注する企業担当者
- 求人情報を探す求職者（副次的）

### 達成目標
- 会社の信頼性・実績を伝え、問い合わせにつなげる
- スマートフォンからの閲覧に最適化する（問い合わせの60%以上がスマホ経由）
- 20万円の予算内で最大限の効果を得る

---

## 2. ページ構成

### 必須ページ（6ページ）

| No. | ページ名 | 主要コンテンツ | 備考 |
|----|---------|-------------|------|
| 1 | **トップページ** | ヒーロー画像、キャッチコピー、事業内容ダイジェスト、施工事例ピックアップ、お問い合わせCTA | ファーストビューで「何をやっている会社か」を即座に伝える |
| 2 | **会社概要** | 会社名・代表者名・所在地・設立年・沿革・建設業許可番号・資格一覧・Google Map | 建設業許可番号は必ず掲載（信頼性の根幹） |
| 3 | **事業内容** | 対応工事の種類（新築・リフォーム・外構・土木等）、業務の流れ | カテゴリごとにアイコン付きで視覚的に整理 |
| 4 | **施工事例** | 事例一覧（写真・工事内容・工期・地域名）、カテゴリ絞り込み | 最重要コンテンツ。ビフォーアフター写真推奨 |
| 5 | **お問い合わせ** | 入力フォーム（氏名・電話・メール・相談内容）、電話番号、受付時間 | フォーム送信 + 電話の二系統を用意 |
| 6 | **プライバシーポリシー** | 個人情報保護方針 | 法的に必須 |

### 推奨追加ページ（予算に余裕があれば）

| ページ名 | 優先度 | 理由 |
|---------|--------|------|
| 代表挨拶 | 高 | 中小企業では「誰がやっているか」が重要な判断材料 |
| お客様の声 | 高 | 第三者評価は信頼獲得に直結 |
| よくある質問（FAQ） | 中 | 問い合わせハードルを下げる |
| 採用情報 | 中 | 建設業は慢性的な人手不足。簡易的でも設置推奨 |
| ブログ/お知らせ | 低 | SEO強化に有効だが、更新体制が必要 |

## 3. デザイン方向性

### カラースキーム
- **メインカラー**: 濃紺（#1a365d）系 — 信頼感・堅実さを表現
- **アクセントカラー**: オレンジ（#e67e22）系 — 活力・行動喚起のCTAに使用
- **ベースカラー**: 白〜ライトグレー — クリーンで読みやすい背景

### デザインコンセプト
- **「安心感 × 地元密着」** を軸にしたデザイン
- 過度に洗練されたデザインよりも、**誠実さ・親しみやすさ**を重視
- 施工写真を大きく使い、実績を視覚的に訴求

### レイアウト・UI要素
- トップページにフルワイドのヒーロー画像（施工現場 or 完成物件）
- スクロールアニメーション（フェードイン程度の軽い演出）
- 画面下部またはヘッダーに**問い合わせボタンを常時固定表示**
- スマホ時はタップで電話発信できるボタンを配置
- 施工事例はカード型レイアウトで一覧表示

### フォント
- 見出し: ゴシック体（Noto Sans JP 等）— 力強く読みやすい
- 本文: ゴシック体 — 可読性重視

## 4. 技術要件

### 制作方式（20万円予算での推奨）
- **WordPress + 建築業向けテーマのカスタマイズ**
- フルオリジナルデザインは予算的に困難。テンプレートベースが現実的

### 必須要件
| 項目 | 内容 |
|------|------|
| レスポンシブ対応 | PC・タブレット・スマートフォン完全対応 |
| SSL対応（HTTPS） | 必須。信頼性とSEOの両面で不可欠 |
| お問い合わせフォーム | Contact Form 7 等で実装 |
| Google Map埋め込み | 会社概要ページに設置 |
| 基本SEO設定 | title・meta description・見出し構造の最適化 |
| OGP設定 | SNSシェア時のサムネイル・説明文 |
| Google Analytics | アクセス解析の導入 |
| ファビコン | ブラウザタブ用のアイコン |

### 推奨要件
- Google Search Console 連携
- サイトマップ自動生成（XML）
- 画像の遅延読み込み（表示速度向上）
- パンくずリスト

## 5. コンテンツ要件

### クライアント提供が必要な素材
| 素材 | 優先度 | 備考 |
|------|--------|------|
| 施工事例の写真（5〜10件分） | 必須 | ビフォーアフターがあるとベスト |
| 会社情報（許可番号、沿革等） | 必須 | 建設業許可番号は必ず |
| 代表者の写真 | 高 | 挨拶文と合わせて信頼感UP |
| ロゴデータ | 高 | なければ簡易制作を検討 |
| スタッフ・現場の写真 | 中 | 親しみやすさの演出に有効 |
| お客様の声テキスト | 中 | 施工事例と紐づけて掲載 |

### 信頼性を高める掲載推奨要素
- 建設業許可番号（ヘッダーまたはフッターに常時表示）
- 創業年数・施工実績件数などの具体的な数字
- 保有資格一覧（一級建築士、施工管理技士等）
- 受賞歴・表彰実績（あれば）
- 「自社施工」であれば強調（中間マージンなしの訴求）

## 9. 注意事項・リスク

- **写真素材の品質がサイトの印象を大きく左右する**。可能であればプロカメラマンの撮影を別途検討
- 20万円はテンプレートカスタマイズが前提。完全オリジナルデザインは予算的に困難
- 公開後の更新・保守体制を事前に決めておくこと（施工事例の追加方法など）
- ブログ/お知らせ機能を入れる場合、更新が滞るとマイナスイメージになるため、更新体制を確認してから導入すること
