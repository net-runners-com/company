## 1. サイトマップ

```
サンプル会社 HP
│
├── [1] トップページ（/index.html）
│       メインビジュアル / 事業概要 / 強み / お知らせ / CTA
│
├── [2] 会社概要（/about.html or セクション内）
│       社名 / 所在地 / 代表者 / 設立 / 事業内容 / アクセスマップ
│
├── [3] 事業内容・サービス紹介（/services.html or セクション内）
│       提供サービス一覧 / 各サービス説明 / 施工実績ギャラリー
│
├── [4] お問い合わせ（/contact.html or セクション内）
│       フォーム（名前・メール・電話・内容）/ 電話番号 / 営業時間
│
└── [5] お知らせ（任意）（/news.html or セクション内）
        新着情報リスト（静的更新）
```

### 構成方針

| 方式 | 採用 | 理由 |
|------|------|------|
| **1ページ完結型（LP型）** | ★推奨 | 予算10万円・静的HTML・5セクション程度であれば、1ページに全セクションを収め、ナビゲーションでスムーズスクロールさせる構成が最もコスパが高い |
| 複数ページ型 | △ | ページ数が増えるとコーディング工数が膨らむ。将来拡張時に移行も可能 |

> **結論**: 1ページ完結型（セクション分割 + スムーズスクロール）を基本とする。お問い合わせのみ、フォームサービスの都合で別ページ化の可能性あり。

## 3. デザインカンプ（デザイン方針）

### 3-1. デザインコンセプト

**「信頼と堅実」** — 建築業にふさわしい、清潔感のある落ち着いたデザイン

| 要素 | 方針 |
|------|------|
| 全体印象 | シンプル・コーポレート・信頼感 |
| 余白 | 十分に確保し、読みやすさを重視 |
| 角丸 | 4〜8px（柔らかさを出しつつ堅実さを保つ） |
| シャドウ | 控えめな box-shadow でカードに立体感 |
| アニメーション | スクロール時のフェードイン程度（軽量に） |

## 4. 技術実装方針

| 項目 | 方針 |
|------|------|
| HTML | セマンティックHTML5、単一ファイル（index.html） |
| CSS | インラインまたは `<style>` タグ内（外部依存ゼロ） |

## 5. ページ別コンテンツ設計

### セクション1: ヒーロー
- **背景**: 建築現場の写真（暗めオーバーレイ `rgba(0,0,0,0.5)`）
- **キャッチコピー**: 「確かな技術で、暮らしを支える。」（仮）
- **サブコピー**: 住宅から商業施設まで、◯年の実績（仮）
- **CTA**: 「お問い合わせはこちら →」

### セクション2: 事業内容
- 3〜4つのサービスカードを横並び（PC）
- 各カード: アイコン + タイトル + 2〜3行の説明文
- 例: 新築工事 / リフォーム / 商業施設 / 外構工事

### セクション3: 施工実績
- グリッドギャラリー（PC: 3列 / SP: 2列）
- 各アイテム: 写真 + 案件名 + 簡単な説明
- 6〜9件程度を表示

### セクション4: 会社の強み
- 3つの強みを大きな数字 + テキストで表現
- 例: 「創業◯年」「施工実績◯件」「リピート率◯%」

### セクション5: 会社概要
- テーブル形式で会社情報を整理
- Google Maps 埋め込み（iframe）

### セクション6: お問い合わせ
- 背景色を Primary（#1B4F72）にして視覚的に差別化
- フォーム: 名前 / メール / 電話 / 種別（ドロップダウン）/ 内容
- 電話番号を大きく表示（SP時はタップで発信）
