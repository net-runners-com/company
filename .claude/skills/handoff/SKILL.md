# セッション引き継ぎ

プロジェクトルート直下の `HANDOFF.md` を以下のフォーマットで作成・更新する:

## フォーマット

```markdown
# セッション引き継ぎ資料

## 完了した作業
- （今回のセッションで完了したタスクをリストアップ）

## 作業中・未完了
- [ ] （途中のタスク。ファイルパス:行番号があれば記載）

## 決定事項
- （重要な設計判断や方針。理由も含める）

## 次回やるべきこと
- （次セッションで最初に取り組むべきこと）

## コンテキスト
- （再開に必要な背景情報。関連ファイル、ブランチ名、エラー状態など）
```

## ルール

1. 会話全体を振り返り、上記フォーマットで `HANDOFF.md` を作成・更新する
2. 具体的に書く（ファイル名、関数名、エラーメッセージなど）
3. 不要な情報は省く（挨拶、経緯の詳細説明など）
4. 既存の HANDOFF.md がある場合は上書きする（最新状態のみ保持）
